Ghi lại toàn bộ thay đổi được thực hiện trong ngày 29/06/2026 cho dự án **E-Learning PHP (MVC)**.

## 1. Tái tạo Database

**File tạo mới:** `e_learning_db.sql`

Do file database gốc bị mất, đã tái tạo lại toàn bộ schema bằng cách đọc ngược từ các Models PHP.

### Các bảng được tạo:
| Bảng | Mô tả |
|------|-------|
| `users` | Tài khoản người dùng, hỗ trợ Google OAuth (`google_id`, `avatar`) |
| `courses` | Thông tin khóa học (`title`, `description`, `thumbnail`, `benefits`, `requirements`) |
| `chapters` | Chương học bên trong khóa học, có `order_num` để sắp xếp |
| `materials` | Tài liệu/bài học trong chương, cột `content` chứa URL hoặc nội dung |
| `enrollments` | Ghi danh học viên vào khóa học (UNIQUE per user+course) |
| `material_completions` | Theo dõi tiến độ học (bài nào đã hoàn thành) |
| `posts` | Bài viết diễn đàn |
| `comments` | Bình luận, hỗ trợ reply lồng nhau qua `parent_id` |

## 2. Bảo mật — Tách credentials ra .env

**Vấn đề:** Credentials Google SSO và thông tin DB đang được hardcode trực tiếp trong source code.

### Thay đổi:
- **Tạo mới** `.env` — chứa toàn bộ biến nhạy cảm
- **Tạo mới** `.env.example` — template chia sẻ cho team (không có giá trị thật)
- **Cập nhật** `.gitignore` — thêm `.env`, `/vendor/`, `/public/uploads/*`
- **Cài đặt** thư viện `vlucas/phpdotenv` v5.6 qua Composer
- **Cập nhật** `public/index.php` — load `.env` bằng `Dotenv\Dotenv::createImmutable()`
- **Cập nhật** `app/config/Database.php` — đọc DB config từ `$_ENV` thay vì hardcode
- **Cập nhật** `app/controllers/AuthController.php` — đọc Google credentials từ `$_ENV`

### Biến môi trường đã tách:
```
DB_HOST, DB_PORT, DB_NAME, DB_USERNAME, DB_PASSWORD
GOOGLE_CLIENT_ID, GOOGLE_CLIENT_SECRET, GOOGLE_REDIRECT_URI
```

## 3. Sửa lỗi (Bug Fixes)

### Bug #1 — `AdminController::storeMaterial()` và `updateMaterial()` — Sai tên cột
- **File:** `app/controllers/AdminController.php`
- **Lỗi:** Dùng `file_url` nhưng bảng `materials` trong DB dùng cột `content`
- **Fix:** Đổi tất cả `file_url` → `content` trong câu SQL INSERT và UPDATE

### Bug #2 — `AdminController::storeCourse()` — Khai báo biến sai vị trí
- **File:** `app/controllers/AdminController.php`
- **Lỗi:** `$benefits` và `$requirements` được khai báo bên trong khối `if (upload ảnh)`, nên nếu không có file upload thì 2 biến này bị `undefined variable`
- **Fix:** Chuyển khai báo ra ngoài trước khối `if`, đảm bảo luôn có giá trị

### Bug #3 — `AdminController::storeChapter()` — Thiếu `order_num`
- **File:** `app/controllers/AdminController.php`
- **Lỗi:** Câu INSERT thiếu cột `order_num`, mọi chapter mới đều bị gán `0`, không sắp xếp được
- **Fix:** Thêm `order_num` vào INSERT, lấy từ `$_POST['order_num'] ?? 0`

### Bug #4 — `EnrollmentController` — Thiếu `require_once Progress.php`
- **File:** `app/controllers/EnrollmentController.php`
- **Lỗi:** Hàm `myCourses()` dùng `new Progress($db)` nhưng class `Progress` chưa được nạp → **Fatal Error**
- **Fix:** Thêm `require_once __DIR__ . '/../models/Progress.php';` ở đầu file

### Bug #5 — `CourseController::show()` — URL redirect sai
- **File:** `app/controllers/CourseController.php`
- **Lỗi:** Redirect về `/public/index.php` (đường dẫn tuyệt đối từ server root) → **404 trên XAMPP**
- **Fix:** Đổi về `?action=home` (relative URL)

### Bug #6 — `LearningController::markDone()` — Thiếu Content-Type JSON
- **File:** `app/controllers/LearningController.php`
- **Lỗi:** Hàm AJAX trả về JSON nhưng không set header → JS có thể không parse đúng
- **Fix:** Thêm `header('Content-Type: application/json');` trước khi echo

## 4. Hoàn thiện tính năng

### 4.1 Đăng nhập / Đăng ký thường (Email + Mật khẩu)

**Trước:** Chỉ có giao diện, không xử lý POST form — chỉ hỗ trợ Google SSO.

**Đã thêm:**
- `User::createUser($fullname, $email, $hashed_password)` trong `app/models/User.php`
- `AuthController::handleLogin()` trong `app/controllers/AuthController.php`
  - Validate email/password
  - Dùng `password_verify()` để so sánh an toàn
  - Lưu session khi đăng nhập thành công
- `AuthController::handleRegister()` trong `app/controllers/AuthController.php`
  - Validate fullname, email, password, password_confirm
  - Kiểm tra email trùng
  - Dùng `password_hash($password, PASSWORD_BCRYPT)` trước khi lưu
- Route `handle_login` và `handle_register` trong `public/index.php`

### 4.2 Admin Dashboard — Số liệu thật từ DB

**Trước:** Dashboard hiển thị số liệu hardcode (12 khóa học, 1,250 học viên, 3,480 ghi danh).

**Đã sửa:**
- `AdminController::dashboard()` — thêm 3 query COUNT thật:
  ```php
  $total_courses     = COUNT(*) FROM courses
  $total_users       = COUNT(*) FROM users WHERE role = 'student'
  $total_enrollments = COUNT(*) FROM enrollments
  ```
- `views/admin/dashboard.php` — thay số hardcode bằng `<?= $total_courses ?>`, `<?= number_format($total_users) ?>`, `<?= number_format($total_enrollments) ?>`

### 4.3 Navbar — Fix link chết

**Trước:** Link "Khóa học" trong navbar (cả desktop và mobile) trỏ vào `href="#"`.

**Đã sửa:** `views/layouts/header.php` — đổi `href="#"` → `href="?action=home"` ở cả 2 vị trí.

## 7. Tích hợp thanh toán VNPAY Sandbox

### 7.1 Cơ sở dữ liệu (Database)
- Bảng `courses`: Thêm cột `price` (INT) mặc định = 0.
- Tạo bảng mới `payments`: Lưu trữ lịch sử giao dịch thanh toán.
  - Các cột: `id`, `user_id`, `course_id`, `amount`, `vnp_txn_ref`, `vnp_transaction_no`, `vnp_response_code`, `status` (pending/success/failed), `created_at`.

### 7.2 Môi trường & Cấu hình
- `.env` và `.env.example`: Bổ sung các biến kết nối VNPAY: `VNP_TMN_CODE`, `VNP_HASH_SECRET`, `VNP_URL`, `VNP_RETURN_URL`.

### 7.3 Logic Code (Controllers & Models)
- **Tạo mới `PaymentController.php`:** Xử lý 2 chức năng chính:
  1. `createPayment()`: Tính toán mã băm bảo mật HMAC SHA512 và redirect user sang trang thanh toán VNPAY Sandbox.
  2. `vnpayReturn()`: Nhận dữ liệu callback từ VNPAY, kiểm tra mã lỗi (00 là thành công) và xác thực chữ ký (vnp_SecureHash). Nếu thành công -> tự động ghi danh (enroll) user vào khóa học.
- **Model `Course.php`:** Cập nhật hàm `createCourse` để lưu trường `price`.
- **`public/index.php`:** Cập nhật router để hỗ trợ `?action=pay` và `?action=vnpay_return`.

### 7.4 Giao diện
- **`views/admin/create_course.php`:** Form tạo khóa học giờ có thêm ô nhập Giá tiền. Thay đổi input thumbnail thành loại file upload.
- **`views/home.php` & `views/courses/detail.php`:** Hiển thị giá tiền thực tế (VD: 500.000đ). Nếu có giá trị > 0, nút bấm chuyển thành "Thanh toán qua VNPAY" thay vì "Đăng ký học ngay".

## 8. Cập nhật bổ sung: Trải nghiệm Google SSO

### 8.1 Sửa luồng Google SSO (Login / Register Intent)
- **File:** `AuthController.php`, `views/auth/login.php`, `views/auth/register.php`.
- **Mục đích:** Phân biệt hành vi của người dùng khi nhấn "Tiếp tục với Google".
- **Thay đổi:**
  - Trang Login: Truyền `intent=login`. Nếu người dùng chưa có tài khoản -> Thông báo bắt buộc phải đăng ký.
  - Trang Đăng ký: Truyền `intent=register`. Nếu người dùng chưa có tài khoản -> Tự động khởi tạo user mới từ dữ liệu Google, sau đó tự đăng nhập.
- Cập nhật hàm `createGoogleUser` trong `User.php` để lưu luôn ảnh đại diện (avatar) lấy từ Google.

## 9. Cập nhật bổ sung: Fix Bug & Cải thiện UI/UX (Cuối ngày)

### 9.1 Sửa lỗi thanh toán VNPAY (Giao dịch hết hạn)
- **Vấn đề:** Môi trường XAMPP mặc định lệch múi giờ, khiến thời gian tạo mã giao dịch bị trễ 7 tiếng so với VNPAY, gây lỗi "Giao dịch đã quá thời gian chờ".
- **Fix:** Ép múi giờ `date_default_timezone_set('Asia/Ho_Chi_Minh');` và bổ sung tham số `vnp_ExpireDate` (hạn 15 phút) vào request gửi lên VNPAY trong `PaymentController.php`.

### 9.2 Sửa lỗi đường dẫn ảnh & thẻ HTML
- **File:** `AdminController.php` và `views/home.php`.
- **Fix lỗi vỡ ảnh:** Sửa đường dẫn upload ảnh trong AdminController từ `/public/uploads/...` thành `uploads/...` (Relative path).
- **Fix hiển thị thẻ HTML thô:** Dùng hàm `strip_tags()` bọc ngoài biến description ở trang chủ để loại bỏ thẻ `<p>` bị in thẳng ra ngoài.

### 9.3 Đồng bộ thay đổi Database (file_url -> content)
- **Vấn đề:** Đã đổi tên cột `file_url` thành `content` trong CSDL nhưng code view chưa update.
- **Fix:** Cập nhật đồng loạt các biến `$material['file_url']` thành `$material['content']` trong:
  - `views/courses/learn.php` (Trang học viên)
  - `views/admin/manage_content.php` (Trang quản lý nội dung Admin)

### 9.4 Fix lỗi Upload File Bài giảng & Dung lượng (Admin)
- **File:** `AdminController.php`
- **Vấn đề:** Hàm `storeMaterial` và `updateMaterial` chưa có logic lưu file upload thật (chỉ lấy tên file), và sẽ bị crash DB (trả về biến $_POST rỗng) nếu user upload file lớn hơn cấu hình `post_max_size` (40MB) của XAMPP.
- **Fix:** 
  - Bổ sung logic `move_uploaded_file()` cho các vật liệu định dạng `type = 'file'`.
  - Bắt lỗi khi biến `$_POST` bị rỗng để hiển thị thông báo "File tải lên quá lớn!" thay vì crash hệ thống.

### 9.5 Nâng cấp Trải nghiệm Học tập (learn.php)

- **UX Click:** Mở rộng vùng bấm (toàn bộ diện tích hàng thay vì chỉ dòng text) để dễ dàng click.
- **Tính năng Toggle Tiến độ (Bật/Tắt):**
